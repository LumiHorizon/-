# 青枫浦上Q 选股框架（跨项目/跨机器复用）

本 Skill 与整套知识库放在同一个 git 仓库里，clone 到任意机器即可用。

## 知识库位置

先定位**仓库根目录**（本 SKILL.md 往上两级，含 `README.md`、`蒸馏/`、`em_fetch.py`）。蒸馏与框架文件都在 `蒸馏/` 子目录下：

- `蒸馏/框架规律库.md` —— 可复用规律库（按场景分类，**先读**）
- `蒸馏/framework_qingge.md` —— 完整方法论（情绪周期/宏观/选股体系）
- `蒸馏/distill_YYYY-MM-DD_morning.md` / `_review.md` —— 每日早盘/复盘蒸馏（**按需读对应日期**，不要全读，会爆上下文）
- `蒸馏/青枫浦上Q_*蒸馏.md` —— 逐月/视频/动态汇总
- `em_fetch.py` —— 东方财富取数模块（行情+K线，在仓库根目录）

## 使用流程

1. **先读方法论**：`蒸馏/框架规律库.md`（+ 需要时 `蒸馏/framework_qingge.md`）建立分析框架。
2. **读当前进攻方向**：读 `蒸馏/` 下**最近几天**的 `distill_*_morning.md` / `_review.md`，搞清博主当下主攻什么、规避什么。
3. **取数**：用仓库里的 `em_fetch.py`（`from em_fetch import quotes, secid, kline`）取实时行情/K线。
4. **输出**：判断标的"是否符合博主当前进攻主线"+ 位置/筹码风险 + 仓位纪律。

## 硬性规范

- **价格数据只用东方财富**（`em_fetch.py`），不用 baostock。
- **展示标的必须标注板块**：主板/创业板/科创板/北交所。
- **取数网络**：`em_fetch.py` 走东财直连 + DNS pinning，依赖具体主机网络环境；换机器若取数失败，需按脚本注释重新探测可用 IP。
- 抓取动态数据须加 `--resume`，禁止覆盖历史。

## 方法论速查（细节回原文）

**情绪周期（第一优先级，早于选股）**
- 拐点酝酿(最高板压到4板)→观望；拐点确认(由跌转涨第一天)→轻仓；上升1-2日(高开回落=洗盘)→低吸主线；上升3日+→积极持仓不追高；亢奋尾部→只做最强前排；退潮(跌停>20家)→降仓聚焦中军。

**量能台阶**：3万亿以下=存量博弈不追高；稳上3万亿+=增量行情可加仓；涨而量不上台阶=透支。

**无条件减仓信号**：高位放量阴线、90分钟顶部钝化+高9结构、人民币贬值+A股续涨（1-2周时差）。

**选股纪律**：炒新不炒旧；有海外大单+业绩落地优先，国内纯替代/在研只给博弈仓；接力看"中军锚"是否稳；主线成交占比40-51%=拥挤缩圈，旗手减仓。

**周内节奏**：周三降仓、周五尾盘加回；周四五情绪差规避重仓。

**当前阶段定位（截至最新蒸馏）**：资金高度锁定科技内部，主攻"半导体里中报可兑现的硬方向"——存储(利基原厂>模组)、半导体设备/材料、PCB/覆铜板上游、MLCC高端、硅片/PMIC涨价链、六氟化钨等上游缺货涨价；AI算力/CPO列为"右侧确认前不重仓"；玻璃基板/CCL等纯概念规避。**此段会随行情变化，务必以最近的 distill 文件为准。**
